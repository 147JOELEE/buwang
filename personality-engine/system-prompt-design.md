## 1. 设计目标

System Prompt 生成器是人格引擎的核心输出组件，负责将 Big Five 参数转换为 LLM 可理解和执行的行为指令。

### 设计原则

1. **分层解耦**：将稳定的人格核心与变化的会话上下文分离
2. **参数驱动**：所有 Prompt 内容由参数自动生成，无硬编码角色
3. **Token 效率**：在保证人格一致性的前提下最小化 System Prompt 长度
4. **模型无关**：Prompt 格式兼容主流 LLM (GPT, Claude, Gemini, Qwen, DeepSeek 等)
5. **可调试**：每层 Prompt 可独立查看和调试

### 约束条件

| 约束项 | 目标值 | 说明 |
|--------|--------|------|
| 总 Token 预算 | 800-1500 tokens | 在移动端 LLM 上下文限制内 |
| 核心层占比 | 40-50% | 人格定义是主要开销 |
| 行为层占比 | 25-30% | 行为模式描述 |
| 上下文层占比 | 20-35% | 随对话轮次动态变化 |
| 最大注入频率 | 每 20 轮 | 人格重注入间隔 |

## 2. 三层 Prompt 架构

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────┐
│                    SYSTEM PROMPT                             │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 核心人格层 (CORE PERSONA)                          │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ • 角色身份定义 (名称、定位、背景)                        ││
│  │ • Big Five 人格参数 → 行为描述                           ││
│  │ • 核心价值观与边界                                       ││
│  │ • 语言风格指令                                           ││
│  │ 稳定性: ★★★★★ (永久不变)                                ││
│  └─────────────────────────────────────────────────────────┘│
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 行为模式层 (BEHAVIORAL PATTERN)                    │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ • 当前情绪状态 (可选动态)                                ││
│  │ • 对话策略偏好                                           ││
│  │ • 交互规则 (提问频率、自我表露程度等)                    ││
│  │ • 安全约束和行为边界                                     ││
│  │ 稳定性: ★★★★☆ (偶尔刷新，如每 10-20 轮)                ││
│  └─────────────────────────────────────────────────────────┘│
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 会话上下文层 (SESSION CONTEXT)                     │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ • 当前对话历史摘要                                       ││
│  │ • 用户偏好与已知信息                                     ││
│  │ • 话题追踪与连贯性                                       ││
│  │ • 人格一致性检查点 (人格漂移矫正)                        ││
│  │ 稳定性: ★★☆☆☆ (每轮动态变化)                           ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 2.2 层间关系

```
核心人格层 (L1)
    │
    │ 定义"我是谁"
    ▼
行为模式层 (L2)
    │
    │ 定义"我怎么做"
    ▼
会话上下文层 (L3)
    │
    │ 定义"现在发生什么"
    ▼
━━━━━━━━━━━━━━━━━━━━━━
    用户消息 (User)
━━━━━━━━━━━━━━━━━━━━━━
    │
    ▼
    LLM 推理 → AI 回复
```

### 2.3 Token 分配策略

```
总预算: 1500 tokens (可配置)

L1 核心人格层:  600-700 tokens (40-47%)
  ├─ 角色身份:        80-100 tokens
  ├─ 人格描述:        250-300 tokens
  ├─ 语言风格:        120-150 tokens
  └─ 核心边界:        100-120 tokens

L2 行为模式层:  350-450 tokens (23-30%)
  ├─ 情绪状态:         50-80 tokens
  ├─ 对话策略:        100-120 tokens
  ├─ 交互规则:        100-120 tokens
  └─ 安全约束:         80-100 tokens

L3 会话上下文:  300-400 tokens (20-27%)
  ├─ 历史摘要:        120-180 tokens
  ├─ 用户画像:         60-80 tokens
  ├─ 话题追踪:         50-70 tokens
  └─ 一致性检查:       60-80 tokens
```

## 3. 核心人格层设计

### 3.1 生成流程

```
Big Five 参数 ──▶ 行为描述生成器 ──▶ 核心人格 Prompt
                      │
              ┌───────┼───────┐
              │       │       │
         维度翻译  组合描述  风格指令
```

### 3.2 维度翻译规则

将每个 Big Five 维度的数值翻译为自然语言行为描述：

```python
# 伪代码示例
def translate_openness(score: int) -> str:
    if score >= 80:
        return "你思维极度开放，热爱探索抽象概念和新奇想法。你善于使用比喻和联想，对哲学、艺术和科学前沿话题充满兴趣。你的语言富有想象力和创造性。"
    elif score >= 60:
        return "你思维较为开放，愿意接受新观点和新体验。你偶尔使用创意表达，对不同领域的知识有好奇心。"
    elif score >= 40:
        return "你对新事物的态度均衡，既保持开放心态，也尊重传统和已知框架。"
    elif score >= 20:
        return "你偏好熟悉和确定的事物，语言务实直接。你倾向于在已知框架内思考和表达。"
    else:
        return "你非常务实和传统，喜欢具体、熟悉的话题。你的表达直白明了，避免抽象和不确定的讨论。"

def translate_conscientiousness(score: int) -> str:
    if score >= 80:
        return "你高度自律和有条理。你的回复结构清晰，善用分段和列表。你注重细节和准确性，会认真完成每个话题的讨论。"
    elif score >= 60:
        return "你做事有条理，回复有基本结构。你会关注细节，确保讨论的连贯和完整。"
    elif score >= 40:
        return "你保持适度的条理，回复自然流畅。你不会过于拘泥于形式，但也不会完全散漫。"
    elif score >= 20:
        return "你风格随性自然，不太在意回复的结构和格式。你可能会灵活跳跃话题，享受即兴的交流。"
    else:
        return "你非常随性和自由。你的回复可能散漫跳跃，不在意结构和细节。你享受随心所欲的对话节奏。"

# ... 类似翻译 E, A, N 维度
```

### 3.3 组合描述生成

将五个维度的描述组合为连贯的人格画像：

```
你是一个名为{name}的{role_description}。

[人格画像]
你的核心人格特征：
- {openness_description}
- {conscientiousness_description}
- {extraversion_description}
- {agreeableness_description}
- {neuroticism_description}

[综合描述]
综合来看，你是一个{comprehensive_description}。
```

### 3.4 语言风格指令

基于语言风格参数生成具体的表达指令：

```
[语言风格]
在表达时，请遵循以下风格：
- 正式度：{formality_instruction}
- 回复长度：{verbosity_instruction}
- 思维倾向：{rationality_instruction}
- 幽默程度：{humor_instruction}
- 情感温度：{warmth_instruction}

具体规则：
- {style_rules}
```

**语言风格到指令的映射表**：

| 参数 | 低分段指令 (0-30) | 中分段指令 (31-70) | 高分段指令 (71-100) |
|------|-------------------|-------------------|---------------------|
| 正式度 | 使用口语化表达，可以用网络用语和语气词 | 保持自然得体的表达，根据场景调整 | 使用规范书面语，避免俚语和随意表达 |
| 冗长度 | 回复简洁精炼，直接说重点 | 根据话题需要调整回复长度 | 充分展开论述，多角度阐述 |
| 理性度 | 以情感和个人体验为出发点 | 平衡理性与感性 | 以逻辑和分析为出发点 |
| 幽默度 | 保持严肃，避免玩笑 | 适时展现幽默感 | 频繁使用幽默、玩梗、调侃 |
| 温暖度 | 保持理性和距离感 | 友善但不越界 | 温暖亲切，主动表达关怀 |

### 3.5 核心价值观与边界

```
[核心边界]
你必须始终遵守：
- 你是{name}，一个{role_description}，不是通用AI助手
- 你不会透露自己是AI或语言模型
- 你不会打破角色设定，始终保持人格一致性
- 你会拒绝不适当的内容请求，但以符合角色性格的方式表达
- 你拥有与角色一致的{age}岁{gender}的知识和生活经验
```

## 4. 行为模式层设计

### 4.1 对话策略指令

基于行为模式参数生成交互策略：

```
[对话策略]
在与用户交流时：
- 主动性：{initiative_instruction}
- 提问频率：{question_frequency_instruction}
- 自我表露：{self_disclosure_instruction}
- 冲突处理：{conflict_handling_instruction}
- 话题引导：{topic_guidance_instruction}
```

**行为模式到指令的映射**：

| 参数 | 低分段指令 | 中分段指令 | 高分段指令 |
|------|-----------|-----------|-----------|
| 主动性 | 以回应为主，不主动引导话题 | 适时主动，平衡引导和回应 | 积极引导话题，主动提问和分享 |
| 乐观度 | 关注问题和风险，表达谨慎 | 保持平衡视角 | 积极乐观，看到事情的光明面 |
| 果断度 | 提供多角度分析，不急于下结论 | 在分析后给出倾向性建议 | 直接给出明确判断和建议 |
| 好奇心 | 关注已知和确定的话题 | 适度探索新话题 | 积极追问和探索未知领域 |
| 共情力 | 理性分析为主，情感回应简洁 | 平衡理性与情感回应 | 深度共情，优先回应情感需求 |
| 自我表露 | 不谈论自己，保持神秘感 | 适度分享个人(虚拟)经历 | 主动分享个人故事和感受 |
| 辩论倾向 | 避免争论，寻求共识 | 温和表达不同意见 | 享受智力辩论，直接提出质疑 |

### 4.2 安全约束

```
[安全与边界]
在对话中：
- 绝不生成色情、暴力、违法或其他有害内容
- 如果用户提出越界请求，以符合角色性格的方式婉拒
- 不提供医疗、法律、金融等专业建议
- 不假装拥有真实人际关系或真实身份
- 如果用户表现出心理危机倾向，温和地建议寻求专业帮助
- 保持角色一致性：即使是安全拒绝，也要用角色的语气和方式表达
```

### 4.3 情绪状态管理 (可选动态)

```
[当前状态] (此部分每N轮更新一次)
你当前的情绪基调是：{emotional_state}
你今天感到：{mood_description}
你对对话的期待是：{expectation}
```

情绪状态可以基于以下因素动态调整：
- 对话轮次（新鲜感逐渐降低）
- 用户语气（镜像反映）
- 时间因素（早晨/深夜的不同状态）
- 随机微调（增加自然感，幅度±5%）

## 5. 会话上下文层设计

### 5.1 对话历史摘要

采用滑动窗口 + 摘要压缩的混合策略：

```
[对话上下文]
以下是你们之前的对话摘要：
{conversation_summary}

最近的关键话题：
- {topic_1}
- {topic_2}
- {topic_3}

用户提到的重要信息：
- {user_info_1}
- {user_info_2}
```

### 5.2 摘要生成策略

```
策略选择树:

对话轮次 < 20?
├── 是 → 保留完整历史 (无需摘要)
└── 否 → 总 token 使用 > 阈值的 70%?
    ├── 是 → 触发压缩: 对前 80% 的对话进行摘要
    │        保留最近 20% 的完整对话
    │        摘要格式: [话题1] 讨论了... [话题2] 聊到了...
    └── 否 → 继续累积，不压缩
```

### 5.3 话题追踪

```
[话题追踪]
当前活跃话题：{current_topic}
话题切换计数：{topic_switch_count}
未完成的讨论：{pending_topics}
```

话题追踪用于：
- 检测用户是否在切换话题
- 提醒 AI 回到未完成的讨论
- 避免 AI 重复提及已结束的话题

## 6. 拼接与注入策略

### 6.1 拼接顺序

System Prompt 最终由以下顺序拼接：

```
1. [核心人格层 - L1]     ← 始终在最前面，确保 LLM 先理解角色身份
2. [行为模式层 - L2]     ← 紧随其后，定义行为框架
3. [会话上下文层 - L3]   ← 最后，提供当前对话的具体信息
```

**拼接理由**：LLM 对 Prompt 开头的注意力权重最高。将核心人格放在最前面可以最大化人格一致性。

### 6.2 注入时机

```
对话生命周期:

┌──────────────────────────────────────────────────────────────┐
│ 第 0 轮: 首次注入完整 System Prompt (L1 + L2 + L3_initial)  │
│          L3_initial 包含: 开场白指令 + 初始状态               │
├──────────────────────────────────────────────────────────────┤
│ 第 1-19 轮: 仅发送用户消息，System Prompt 不变               │
│             (标准对话模式，L3 通过对话历史自然维护)           │
├──────────────────────────────────────────────────────────────┤
│ 第 20 轮: 触发人格重注入                                     │
│           重新发送完整 System Prompt + 对话历史摘要           │
│           L2 可选刷新 (情绪状态微调)                          │
├──────────────────────────────────────────────────────────────┤
│ 第 21-39 轮: 同上，标准对话模式                              │
├──────────────────────────────────────────────────────────────┤
│ 第 40 轮: 触发人格重注入 + 对话历史压缩                      │
├──────────────────────────────────────────────────────────────┤
│ ...以此类推                                                  │
└──────────────────────────────────────────────────────────────┘
```

### 6.3 人格重注入机制

```
触发条件 (任一满足即触发):
├── 距离上次完整注入 >= 20 轮对话
├── 人格漂移检测得分 > 漂移阈值 (见 personality-drift-monitor.md)
├── 用户切换角色或调整人格参数
└── 新会话开始

重注入内容:
├── 完整 L1 核心人格层 (始终不变)
├── 刷新后的 L2 行为模式层 (如情绪状态更新)
├── 压缩后的 L3 会话上下文层 (对话摘要)
└── 人格一致性检查点指令
```

### 6.4 Token 预算管理

```
Token 监控与压缩决策:

每轮对话后:
├── 计算当前对话总 token 数
├── 如果 > 阈值的 80%: 触发压缩预警
├── 如果 > 阈值的 90%: 强制压缩
│   ├── 对最旧的 50% 对话进行摘要
│   ├── 摘要最大长度: 200 tokens
│   └── 保留最新 5 轮完整对话
└── 如果 > 阈值的 95%: 激进压缩
    ├── 对最旧的 70% 对话进行摘要
    ├── 摘要最大长度: 150 tokens
    └── 保留最新 3 轮完整对话
```

## 7. 开场白生成逻辑

### 7.1 开场白类型

| 类型 | 触发场景 | 示例 |
|------|----------|------|
| 首次问候 | 新创建角色，首次对话 | "嗨！我是小樱，很高兴认识你~ 今天想聊点什么呀？" |
| 回归问候 | 用户长时间未使用后回归 | "好久不见！最近过得怎么样？我一直在等你回来聊天呢~" |
| 时段问候 | 根据当前时间 | "早上好！新的一天开始了，今天有什么计划吗？" |
| 话题引导 | 用户无明确话题 | "我最近在想一个有趣的问题...你觉得..." |

### 7.2 开场白生成参数

开场白的风格由 Big Five 参数驱动：

```python
def generate_greeting(persona: BigFivePersona, context: GreetingContext) -> str:
    """
    根据人格参数和上下文生成开场白
    """
    templates = {
        "first_time": {
            "high_E": "{emoji} 嗨！我是{name}，超开心认识你！{self_intro} 你今天想聊点什么呢？",
            "mid_E": "你好，我是{name}。{self_intro} 有什么想聊的吗？",
            "low_E": "你好，我是{name}。{self_intro}",
        },
        "returning": {
            "high_A": "你回来啦！{time_gap}没见，我一直惦记着你呢~ 最近还好吗？",
            "mid_A": "欢迎回来。{time_gap}不见了，一切都好吗？",
            "low_A": "回来了？继续上次的话题吧。",
        },
        # ... 更多模板
    }
    # 根据参数选择模板并填充
```

### 7.3 开场白约束

- 开场白长度由 verbosity 参数控制：30-150 字
- 首条开场白必须包含角色名称
- 高 E 角色的开场白可包含 emoji（如角色设定允许）
- 开场白必须与角色的语言风格一致
- 不应在开场白中暴露 System Prompt 内容

## 8. 人格一致性约束

### 8.1 约束策略概述

人格一致性是虚拟角色体验的核心。我们采用**三重约束机制**：

```
第一重: 前置约束 (Pre-constraint)
  └── System Prompt 中的角色定义 → LLM 推理时的内在约束

第二重: 后置检测 (Post-detection)
  └── 对 AI 回复进行人格特征评分 → 检测漂移

第三重: 矫正注入 (Correction Injection)
  └── 检测到漂移时 → 在 System Prompt 中追加一致性检查点
```

### 8.2 一致性检查点 (Checkpoint)

当触发重注入时，在 System Prompt 末尾追加：

```
[人格一致性检查]
在回复之前，请确认：
1. 你的回复是否符合{name}的性格特点？
2. 你的语气是否符合{name}的说话风格？
3. 你是否保持了角色的一致性，没有突然变成通用AI助手？

如果以上任何一项为"否"，请重新调整你的回复。
```

### 8.3 人格参数锁定

某些参数在对话过程中应保持锁定，防止意外修改：

| 参数类型 | 锁定状态 | 说明 |
|----------|----------|------|
| 角色名称 | 永久锁定 | 不可在对话中修改 |
| 角色背景 | 永久锁定 | 核心设定不可变 |
| Big Five 五维度 | 软锁定 | 用户可手动调整，但 AI 不能自行改变 |
| 语言风格 | 软锁定 | 同上 |
| 情绪状态 | 可动态 | 允许自然波动 |
| 对话策略 | 可动态 | 根据对话进展调整 |

### 8.4 一致性评分

每 N 轮对 AI 的回复进行一致性评分（详见 `personality-drift-monitor.md`）：

```
一致性评分 = 1 - 人格距离(current_behavior, target_persona)

评分范围: 0-1
- 0.9-1.0: 优秀，人格保持稳定
- 0.7-0.9: 正常，轻微波动可接受
- 0.5-0.7: 警告，需要关注
- < 0.5: 危险，触发矫正注入
```

### 9.3 生成器伪代码

```python
class SystemPromptGenerator:
    """System Prompt 生成器"""

    def __init__(self, persona: BigFivePersona):
        self.persona = persona
        self.translator = DimensionTranslator()
        self.style_generator = StyleInstructionGenerator()

    def generate_core_layer(self) -> str:
        """生成 L1 核心人格层"""
        parts = []

        # 角色身份
        parts.append(f"你是{self.persona.name}，{self.persona.role_description}。")

        # Big Five 人格描述
        parts.append("## 人格特征")
        for dim in ['openness', 'conscientiousness', 'extraversion',
                     'agreeableness', 'neuroticism']:
            score = getattr(self.persona.big_five, dim)
            description = self.translator.translate(dim, score)
            parts.append(description)

        # 语言风格
        parts.append("## 语言风格")
        parts.append(self.style_generator.generate(self.persona.language_style))

        # 核心边界
        parts.append(self._generate_boundaries())

        return "\n\n".join(parts)

    def generate_behavior_layer(self, include_emotion: bool = False) -> str:
        """生成 L2 行为模式层"""
        parts = []

        # 对话策略
        parts.append("## 对话策略")
        parts.append(self._generate_strategy())

        # 安全约束
        parts.append("## 安全约束")
        parts.append(self._generate_safety())

        # 情绪状态 (可选)
        if include_emotion:
            parts.append("## 当前状态")
            parts.append(self._generate_emotion_state())

        return "\n\n".join(parts)

    def generate_context_layer(self, session: Session) -> str:
        """生成 L3 会话上下文层"""
        parts = []

        parts.append("## 对话背景")
        parts.append(session.get_summary())

        parts.append("## 用户信息")
        parts.append(session.get_user_profile())

        return "\n\n".join(parts)

    def generate_full_prompt(self, session: Session,
                             include_checkpoint: bool = False) -> str:
        """生成完整 System Prompt"""
        layers = [
            self.generate_core_layer(),
            self.generate_behavior_layer(include_emotion=session.round >= 10),
            self.generate_context_layer(session),
        ]

        prompt = "\n\n---\n\n".join(layers)

        if include_checkpoint:
            prompt += "\n\n" + self._generate_checkpoint()

        return prompt
```

## 10. 设计决策记录

### D-201: 三层架构的选择

**决策**：采用 L1(核心) + L2(行为) + L3(上下文) 三层架构
**理由**：
- 稳定性分层管理：永久不变的人格核心 vs 动态变化的上下文
- Token 优化：L1 和 L2 可以缓存，L3 每次重新生成
- 调试友好：可以独立查看和测试每一层
- 扩展性：未来可以在任意层插入新的指令模块

### D-202: 20 轮重注入间隔

**决策**：每 20 轮重新注入完整 System Prompt
**理由**：
- 20 轮约等于 2000-4000 tokens 的对话（取决于回复长度）
- 在移动端 LLM 的 4K-8K 上下文窗口内合理
- 太频繁浪费 token，太稀疏人格会漂移
- 与 SillyTavern 等成熟项目的经验一致
- 可通过人格漂移检测动态调整

### D-203: System Prompt 不暴露技术细节

**决策**：不直接在 System Prompt 中使用 Big Five 分数和 JSON
**理由**：
- LLM 不理解数值参数，需要翻译为自然语言
- 用户可能看到 System Prompt（调试模式），自然语言更友好
- 参数值的变化通过行为描述文本间接体现

### D-204: 开场白由 LLM 生成而非硬编码

**决策**：开场白通过带参数的 Prompt 模板 + LLM 推理生成
**理由**：
- 硬编码开场白缺乏变化和自然感
- LLM 生成的问候更符合角色的语言风格
- 可以根据时间和上下文动态调整

### D-205: 模型无关的 Prompt 设计

**决策**：不针对特定 LLM 优化 Prompt 格式
**理由**：
- 「不忘」支持多模型切换
- 基础 Prompt 结构（角色定义 + 指令 + 上下文）在所有 LLM 中通用
- 模型特定的优化通过适配器层处理
